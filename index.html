第二階段 · 核心定義

從需求分析到
功能規劃

AI 擔任「產品經理 PM」——有了洞察後，確保開發的東西是市場要的

  • 分析需求
    評估哪些需求最重要，將需求轉化成功能
  • 架構設計
    評估技術線、繪製系統圖、提出 tradeoffs
  • User Stories 與 PRD 生成
    AI 撰寫完整且邏輯嚴密的開發規格 spec，避免溝通落差
  • 功能地圖 Sitemap / Flow
    快速產出功能架構與邏輯流程
AI 代理的角色

產品經理 (PM)

核心價值

將模糊的使用者洞察轉化為可執行的開發規格，讓工程師與設計師有共同的語言

第三階段 · 開發執行

AI 扮演
你的全端工程師

MVP 的快速產出與迭代邏輯

  • 架構設計 — 評估技術選型、繪製系統圖、提出 tradeoffs
  • 程式生成 — 依 spec 產出可運行的程式碼，不只是片段
  • Code Review — 自動指出潛在 bug、安全漏洞、效能問題
  • 測試撰寫 — 根據 AC 生成單元測試、整合測試
  • 文件同步 — 隨程式碼更新自動補全 README / API doc

第三階段 · 開發執行

不追求完美，
追求「快」且「準」的落地

  • 從 UserFlow 到 Wireframe
    利用 AI 快速草擬原型介面，驗證流程順暢度
  • MVP 的取捨藝術
    詢問 AI「哪些功能是首版必須的？」以達成最小可行性
  • 程式生成
    依 spec 產出可運行的程式碼
快速迭代與反饋迴圈

數據驅動——根據首版投放後的反饋數據，由 AI 生成優化策略建議
